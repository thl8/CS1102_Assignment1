What are Floating-Point Numbers?

A floating-point number is defined as Fraction(F) * Radix(R) ^ Exponent (E). Since computers use binary forms to represent them, so they are shown as F*2^E. There are 2 interpretation strategies: 32-bit and 64-bit single-precision. Number 0 cannot be expressed in normalized form. There are normalized form and de-normalized form due to normalized form constraints.

⚠ BE CAUTION, due to finite bits, floating-point numbers may not represent preciously.

How Do Computers Handles Floating-Point Numbers?

32-bit single-precision

A floating-point number is displayed by 1 sign bit (S), followed by 8 bits of E, and the remaining 23 bits of F. Sign bit can only be 0 for positive number and 1 for negative number.

Normalized Form

Sign bit Exponent Fraction
0 01111111 10000000000000000000000
  1. Sign bit = 0, therefore it is a positive number.
  2. Exponent = 01111111. The real exponent is E minus 127. In this case, we first convert 01111111b into decimal form, which is 127. Then, real exponent would be 127 - 127 = 0
  3. Fraction = 10000000000000000000000. The real fraction would be 1.F. In this case, the real fraction = 1+1*2^(-1) = 1.5
  4. Computer representation for 1.5*2^0. (1.5D)

De-normalized Form

Sign bit Exponent Fraction
1 00000000 11110000000000000000000
  1. Sign bit = 1, therefore it is a negative number.
  2. Exponent = 0. The real exponent must be -126
  3. Fraction = 11110000000000000000000. The real fraction would be 0.F. In this case, the real fraction = 1*2^(-1)+1*2^(-2)+1*2^(-3)+1*2^(-4) = 0.9375.
  4. Computer representation for -0.9375*2^-126. (-1.1020259539e-38D)

64-bit single-precision

A floating-point number is displayed by 1 sign bit (S), followed by 11 bits of E, and the remaining 52 bits of F. Compared to 32-bit, 64-bit single precision could make floating-point numbers more precious.

Normalized form

Sign bit Exponent Fraction
0 01111 11111 1 00000 00000 00000 00000 00000 00000 00000 00000 00000 00000 00
  1. Sign bit = 0, therefore it is a positive number.
  2. Exponent = 01111 11111 1. The real exponent is E minus 1023. In this case, we first convert 01111111111b into decimal form, which is 1023. Then, real exponent would be 1023 - 1023 = -0
  3. Fraction=00000 00000 00000 00000 00000 00000 00000 00000 00000 00000 00. The real fraction would be 1.F. In this case, the real fraction = 1
  4. Computer representation for 1*2^0. (1D)

De-normalized form

Sign bit Exponent Fraction
1 00000000000 10000 00000 00000 00000 00000 00001 00000 00000 00000 00000 01
  1. Sign bit = 1, therefore it is a negative number.
  2. Exponent=0. The real exponent must be -1022.
  3. Fraction=10000 00000 00000 00000 00000 00001 00000 00000 00000 00000 01. The real fraction would be 0.F. In this case, the real fraction = 1*2^(-1)+ 1*2^(-30)+1*2^(-52) = 0.5.
  4. Computer representation for -0.5*2^-1022. (-1.11253693132586270022528594514E-308D)
